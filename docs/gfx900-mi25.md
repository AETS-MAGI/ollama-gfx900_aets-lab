# gfx900 / MI25 Guide

[English](./gfx900-mi25.md) | [日本語](./gfx900-mi25.ja.md)

This document describes build and runtime behavior specific to AMD Vega / gfx900 /
Radeon Instinct MI25 in this fork.

> **Scope:** This fork targets gfx900 inference on Linux with ROCm.
> It is not an official compatibility statement for MI25/gfx900.
> Claims in this document are limited to what is directly verifiable from source code
> and worklog evidence.

---

## 1. What This Fork Does

Upstream Ollama blocks gfx900/gfx906 execution (commit `55ca82726`).
This fork removes that block and adds targeted optimizations for gfx900:

| Change | File | Status |
|--------|------|--------|
| `IsROCmGFX900()` detection method added | `ml/device.go` | code-verified |
| Flash Attention disabled for gfx900 | `ml/device.go:FlashAttentionSupported()` | code-verified |
| `num_parallel=1` forced for gfx900 | `server/sched.go` | code-verified |
| Default `num_ctx` capped at 8192 for gfx900 | `server/routes.go` | code-verified |
| `build-gfx900/lib/ollama` prioritized in path search | `ml/path.go` | code-verified |
| `gfx900` added to `CMakePresets.json` ROCm 6 and ROCm 7 | `CMakePresets.json` | code-verified |
| Dockerfile rocBLAS cleanup narrowed to `*gfx906*` (preserving gfx900) | `Dockerfile` | code-verified |

## 2. Prerequisites

- Linux (Ubuntu 22.04 or 24.04 recommended)
- ROCm 6.1 or later installed (`amdgpu-install`)
- `rocminfo` shows `gfx900` device
- Go 1.24.1+
- CMake 3.21+, HIP compiler (from ROCm)

Verify your device is detected:

```bash
rocminfo | grep -E "Name|gfx"
```

Expected output includes a line like:
```
  Name:                    gfx900
```

## 3. Building

### 3.1 Build Ollama for gfx900

Use the `vega-int8-probe` branch, which contains all gfx900-specific patches:

```bash
git checkout vega-int8-probe
```

Configure and build with explicit gfx900 target:

```bash
cmake -B build-gfx900 \
    -DAMDGPU_TARGETS=gfx900 \
    -DCMAKE_BUILD_TYPE=Release
cmake --build build-gfx900 --parallel $(nproc)
```

Build artifacts are placed in `build-gfx900/lib/ollama/`.
The runtime path search in `ml/path.go` prioritizes this directory automatically.

### 3.2 Build rocBLAS for gfx900 (if needed)

The system ROCm package may not include gfx900-specific Tensile kernels.
If rocBLAS initialization fails or performance is degraded, build locally:

```bash
# See build-rocblas-gfx900.sh in the repository root
./build-rocblas-gfx900.sh
```

Then inject the local library path at runtime:

```bash
export ROCBLAS_TENSILE_LIBPATH=/path/to/local/rocblas/library
```

## 4. Runtime Behavior

### 4.1 Automatic gfx900 detection

When a ROCm device with `compute=gfx900` is detected, the following are applied
automatically at server startup — no manual environment variables required:

- **Flash Attention**: disabled (`ml/device.go`)
- **`num_parallel`**: forced to 1 (`server/sched.go`)
- **`num_ctx` default**: capped at 8192 (`server/routes.go`)

Log output confirming gfx900 detection:

```
level=INFO msg="forcing num_parallel=1 for ROCm gfx900" gpu=... compute=gfx900
level=INFO msg="capping default context for ROCm gfx900" default_num_ctx=8192
```

### 4.2 Device visibility

To restrict Ollama to the MI25:

```bash
export ROCR_VISIBLE_DEVICES=<UUID>
```

Get the UUID with:

```bash
rocminfo | grep -A5 "gfx900" | grep "Uuid"
```

### 4.3 Library path

At dev time, the runtime searches for backends in this order (`ml/path.go`):

1. `<exe-dir>/build-gfx900/lib/ollama`
2. `<cwd>/build-gfx900/lib/ollama`
3. Distribution paths (`../lib/ollama`, etc.)
4. `<exe-dir>/build/lib/ollama`
5. `<cwd>/build/lib/ollama`

To override explicitly:

```bash
export OLLAMA_LIBRARY_PATH=/your/custom/path
```

## 5. Known Limitations

| Limitation | Reason | Source |
|------------|--------|--------|
| Flash Attention not available | Excluded for gfx900 stability | `ml/device.go:FlashAttentionSupported()` |
| Max concurrent requests: 1 | `num_parallel` forced to 1 | `server/sched.go` |
| Effective context window: ≤ 8192 tokens (default) | `num_ctx` capped | `server/routes.go` |
| Windows not supported | Upstream Windows HIP excludes gfx900 | `CMakeLists.txt` |
| MLIR iGEMM not available | Disabled upstream for gfx900 (commit `2407d2f`, 2021) | MIOpen source |
| rocBLAS Tensile kernels may be absent | Not included in standard ROCm packages for gfx900 | runtime-observed |
