## AMD Radeon

### gfx900 / MI25（このフォーク）

このフォークは AMD Vega / gfx900 / Radeon Instinct MI25 向けの明示的なサポートを追加します。
gfx900 は **upstream Ollama の AMD 対応表には含まれておらず**、フォーク固有のビルドが必要です。
コードで確認済みのgfx900向け挙動制御は以下の通りです：

| 挙動 | 値 | 根拠（コード） |
|------|----|----|
| Flash Attention | 使用されない — `FlashAttentionSupported()` が `false` を返す。`OLLAMA_FLASH_ATTENTION` は無効 | `ml/device.go` |
| デフォルト `num_parallel` | 1（強制） | `server/sched.go` |
| `num_ctx` 上限 | 8192 — VRAM 基準のデフォルトが 8192 超の場合のみ適用。MI25（16 GB）ではデフォルト通常 4096、キャップ不適用 | `server/routes.go` |
| ビルド出力ディレクトリ | `build-gfx900/lib/ollama` | `ml/path.go`（優先探索） |

詳細なビルド・実行手順は [gfx900 / MI25 ガイド](./gfx900-mi25.ja.md) を参照してください。

### Linux でのオーバーライド

Ollama は AMD ROCm ライブラリを使用しますが、すべての AMD GPU がサポートされているわけではありません。
一部のケースでは、近いアーキテクチャをターゲットとして強制することができます。

例: Radeon RX 5400 は `gfx1034` ですが ROCm の直接サポートがありません。
最も近い `gfx1030` をターゲットにするには：

```bash
export HSA_OVERRIDE_GFX_VERSION="10.3.0"
```

> **gfx900 注記:** このフォークでは `gfx900` をネイティブターゲットとしてビルドするため、
> gfx900/MI25 に対しては `HSA_OVERRIDE_GFX_VERSION` の設定は**不要**です。

既知の LLVM ターゲットと GPU の対応：

| LLVM ターゲット | GPU 例 |
|---|---|
| gfx900 | Radeon Instinct MI25（**このフォーク対象**） |
| gfx908 | Radeon Instinct MI100 |
| gfx90a | Radeon Instinct MI210/MI250 |
| gfx942 | Radeon Instinct MI300X/MI300A |
| gfx1010 | Radeon RX 5700 XT |
| gfx1030 | Radeon PRO V620 |
| gfx1100 | Radeon PRO W7900 |
| gfx1102 | Radeon RX 7600 |
| gfx1200 | Radeon RX 9070 |
| gfx1201 | Radeon RX 9070 XT |

### GPU の選択

複数の AMD GPU があり、Ollama が使用する GPU を絞りたい場合は `ROCR_VISIBLE_DEVICES` を設定します。

```bash
export ROCR_VISIBLE_DEVICES=<GPU-UUID>
```

デバイス一覧は `rocminfo` で確認できます。

## Metal（Apple GPU）

Ollama は Metal API を介して Apple デバイスの GPU アクセラレーションをサポートします。

## Vulkan GPU サポート

> **注記:** Vulkan は現在実験的な機能です。有効にするには `OLLAMA_VULKAN=1` を設定してください。

Windows および Linux 上で [Vulkan](https://www.vulkan.org/) による追加 GPU サポートが利用可能です。

```bash
sudo setcap cap_perfmon+ep /usr/local/bin/ollama
```
